Git

一、git

  1. 官网：http://git-scm.com/

  2. Linux系统（另有BitKeepper，但是收费）

  3. 版本控制系统：

    • 本地版本控制系统：每个计算机自带的多个版本。若计算机之间需要相互传送文件，则需要借助服务器(gitHub或gitLab。gitHub又分为public和private，public免费，private收费；gitLab在公司内部发布)。

    • 集中化版本控制系统（如：cvs svn）

    • 分布式版本控制系统（如：git）

  4. 优势：

    • 控制本地版本：全量（而svn只是增量），即每次你修改了一个版本的文件，所有的版本文件都会被重新记录（而svn只会记录修改了的版本文件），即可以时刻保持数据的完整性。
    • 重写提交说明：只改变说明。
    • 可以后退：几乎每步操作都可以回滚。
    • 分支系统：对稳定性和性能的不同要求。

二、文件基操

  1. 三个区域（知识点）：

    工作区：存放git所管理的文件，而git不管理的文件是脱离的，文件是红色的；

    暂存区：执行add命令后的文件存放的地方，文件是绿色的；

    对象区：执行commit命令的文件存放的地方，文件等待被push给gitHub服务器；

  2. 文件的三种状态（知识点）：

    已修改(modified)

    已暂存(staged)

    已提交(commited)

  3. 对文件进行操作的常用的命令：

  • ==git init==作用：若本地无版本控制，即.git文件，则会生成一个给你，同时生成master分支，还有指向master的一个指针HEAD，init后该文件接受git管理；

  • ==rm -rf .git==作用：会删除你的版本控制，即.git文件，同时master也没了；

  • ==touch 文件名.后缀==作用：会在Git文件下生成相应文件，文件保存在工作区；

  • ==git status==作用：功能强大。会反馈给你：当前git识别的文件所在的区域、以及可对它进行的操作。按照你每次工作的文件分开反馈，时间越早的文件越在下方；

  • ==git mv 文件名1.后缀 文件名2.后缀==作用：用git实现rename: 文件.后缀->文件2.后缀，名字变为文件名2；

    若撤销重命名的操作，这个文件名写的是文件名1，这步操作会把文件1以git删除方式删除，即重命名后产生了一个内容一样的文件2，它的名字和修改的名字保持一致；

  • ==mv 文件名1.后缀 文件名2.后缀==作用：用系统实现rename: 文件.后缀->文件2.后缀，名字变为文件名2；

  • ==git add 文件名.后缀==作用：把touch命令生成的文件放在暂存区，git status反馈：

  • ==git add .==作用：把touch生成的所有文件放到暂存区；

  • ==toch .gitignore==作用：生成.gitignore文件

    ==vi .gitignore==作用：输入要忽略的文件名.后缀，保存退出，就可以在commit时忽略这些文件，如果是一个空目录则会自动忽略；

    文件夹名/（文件夹）、文件夹名/星号.后缀（文件夹下的文件）、文件夹名/星号/星号.后缀（文件夹下的文件夹下的文件）都可以作为被忽略的对象；

  • ==git commit -m “提交的说明内容”==作用：将暂存区的文件放入对象区，同时会反馈提交文件的信息:

  • ==git commit –amend -m “说明内容”==作用：把最近的一次提交的说明内容重置；

  • ==git log==作用：反馈每次提交的信息：

  • ==git log –pretty=oneline==作用：按照=oneline的格式输出日志；

  • ==git log –pretty=format:”%h - %an ,%ar : %s”==作用：按照=format:”%h - %an ,%ar : %s”的格式输出日志；

  • ==git log -数字==作用：反馈最近两次提交的信息；

  • ==vi 文件名.后缀==作用：

    在弹出的窗口，按a进入插入模式，可以进行编辑；

    ==:set number==作用：每行的数字会显示出来，该命令要退出插入模式，在下面输入；

    ==:数字==作用：如数字9，光标就会跳到第9行，该命令要退出插入模式，在下面输入；

    如果下面出现一个:你按q退出去，就会回到刚刚的模式下；

    完成编辑后，按Esc退出插入模式，按Shift+zz保存并退出vi；

    vi后就把文件从对象区放到工作区了。

  • ==git checkout – 文件名.后缀==作用：对相应文件，撤销最近执行的动作命令；

  • ==git rm –cached 文件名.后缀==作用：将文件从暂存区退回工作区；

  • ==git reset head 文件名.后缀==作用：会把你新建的放在了暂存区的文件放回工作区；

  • ==git rm 文件名.后缀==作用：用git删除在对象区里相应的文件，会把文件放到暂存区；

    若你不想删除它了，就将文件先放回工作区，再输入取消删除动作的命令；

    若继续执行==git commit -m “说明内容”==，会把文件彻底删除；

  • ==rm abc.txt==

    ==git add abc.txt==

    ==git commit -m “提交的说明内容”==作用：用操作系统来删已提交的文件，会把文件放到工作区，再add，再commit就删掉了；

  • ==echo ‘添加内容’ > 文件名.后缀==作用：给相应的文件添加内容；

  • ==cat 文件名.后缀==作用：反馈相应文件的内容；

  • ==cd ..==作用：返回文件的上一层；

  • ==git reset HEAD 文件名.后缀==

    ==git checkout – 文件名.后缀==作用：第一句命令将文件放回工作区，第二句命令将文件身上的动作撤销；

  • ==通配符：*.后缀==作用：指定所有的这样的文件；

    ==通配符：!文件名.后缀==作用：否定所有这样的文件；

  • ==Ctrl+c==作用：结束本次命令，进入下一个命令；

  • ==Ctrl+a==作用：使光标飞到开头；

  • ==Ctrl+e==作用：使光标飞到末尾；

  • ==clear或Ctrl+L==作用：清除窗口内所有内容；

  • ==unknow==作用：提示命令格式不对；

  • ==修改用户，在git文件中打开config文件修改你的用户名和邮箱，在修改后，在修改前提交的文件的用户会变成该用户==

三、用户名和邮箱

  1. 要操作git必备的两样东西

  2. 设置用户名和邮箱的3种方法：

  • git config –global（优先级最低，给整个计算机一次性设置）

  • git config –system（优先级次之，给当前计算机系统用户一次性设置，推荐）

  • git config –local（优先级最高，给当前项目一次性设置）

  1. 常用命令：

  • ==pwd==作用：进入对用户名和邮箱的操作；

  • ==cd .git/==作用：将目录转进.git文件下;

  • ==ls==作用：查看当前目录下的所有文件；

  • ==git config –local user.name ‘你的用户名’==作用：给当前项目添加用户名；

  • ==git config –local user.email ‘你的邮箱’==作用：给当前项目添加邮箱；

  • ==cat config==作用：查看–local设置后的config文件中的配置；

  • ==git config –system user.name ‘你的用户名’==作用：给计算机系统用户添加用户名；

  • ==git config –system user.email ‘你的邮箱’==作用：给计算机系统用户添加邮箱；

  • ==cat .gitconfig==作用：查看–system设置后的.gitconfig文件中的配置；

  • ==git config –local –unset user.name==作用：在config文件中删除了–local时建立的user.name；

  • ==git config –local –unset user.email==作用：在config文件中删除了–local时建立的user.email；

四、分支操作

开发到一般的时候，会打分支，如：branch是一个总分支，分成master和xxx两个分支；

每个分支都有总分支branch的所有文件；

每个又是独立的，若给master分支添加一个文件，xxx分支中不会有文件；

git目录下的文件会随着当前工作区而变，会显示对象区的文件，以及正在进行vi的文件。

  1. 操作分支的命令：

    ==git branch==作用：查看分支

    ==git branch 名字==作用：创建该名字的分支

    ==git checkout 分支名==作用：切换到该名字的分支

    ==git cherckout -b 分支名==作用：创建并切换到该分支

    ==git branch -d 分支名==作用：删除改名字的分支，注意：不能删除当前的分支；若那你要删除的分支含有独有的文件，建议把独有的文件合并merged到你当前的分支下，这样就不会报错；

    ==git merge 分支名==作用：将该分支的文件合并到当前的分支下

    ==git checkout -D 分支名==作用：强行删除分支

    ==git branch -v==作用：每个分支最近提交的信息，shal和”提交说明“

    ==cat HEAD==作用：查看head指向的分支名

    ==git log –graph==作用：查看图形化日志
