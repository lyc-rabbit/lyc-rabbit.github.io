一、Spring Boot入门

  1. 简介

    • 背景：

      J2EE笨重的开发、繁多的配置、低下的开发效率、复杂的部署流程、第三方技术集成难度大。

    • 解决：

      “Spring全家桶”时代：

      Spring Boot –>J2EE一站式解决方案

      Spring Cloud –>分布式整体解决方案

    • 优点：

      • 快速创建独立运行的Spring项目以及与主流框架集成
      • 使用嵌入式的Servlet容器，应用无需打包成WAR包
      • starters自动依赖与版本控制
      • 大量的自动配置，简化开发，也可修改默认值
      • 无需配置XML，无代码生成，开箱即用
      • 准生产环境的运行时应用监控
      • 与云计算的天然集成

    • 缺点：需要非常了解API

  2. 微服务

    • 微服务

      2014，Martin Fowler

      微服务：架构风格

      一个应用应该是一组小型服务；

      可以通过HTTP的方式进行互通；

      一个微服务架构把每个功能元素放进一个独立的服务中，每一个功能元素最终都是一个可独立替换和独立升级的软件单元…并且通过跨服务器分发这些服务器进行扩展，只在需要时才复制；

  • 单体应用：ALL IN ONE；

    一个单体应用程序把它所有的功能放在一个单一进程中…并且通过在多个服务器上复制这个单体进行扩展；

  1. 原理分析

    1. 父项目

      Spring Boot 的版本的仲裁中心：

      ​ 即，以后导入依赖默认是不需要写版本号的（除非导入的依赖，没有在dependencies里面被管理）。

    2. spring-boot-starter-web

      • Spring Boot 将所有的场景都抽取出来，做成一个一个的starters（启动器）。只需要在项目里面引入这些功能相应的starters的相关场景，该场景的所有依赖都会被导入进来，版本由spring自动控制。

      • spring-boot-starter-：spring-boot==场景启动器==；

      • spring-boot-starter-web：用来导入web模块，正常运行所依赖的组件；

      • spring-boot-starter-amqp：高级消息队列

      • spring-boot-starter-aop：aop

      • spring-boot-starter-data-jpa：数据库jpa访问

      • spring-boot-starter-data-redis：数据库redis模块

      • spring-boot-starter-mail：邮件开发

      • spring-boot-starter-web：web开发

      • 场景 同行的 POM 中介绍了该场景会导入的 jar
